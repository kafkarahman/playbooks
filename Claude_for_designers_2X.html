Claude for Designers

A practical, experiment-first guide for 2X designers across Creative, UI/UX, and Video — showing exactly how Claude fits into real work, with real examples and ready-to-use prompts. Version 1, June 2026.

Why this guide exists

Claude is not a magic button. It will not replace your design instinct, your client relationships, or your taste. What it will do is remove the repetitive, time-consuming work that gets between you and the parts of the job you actually want to do.

This guide was built from real case studies — documented workflows from designers who are already using Claude on real projects — plus hands-on research into every tool and connector available as of June 2026.

How to Use This Guide

Jump straight to your pillar using the left menu. Each pillar starts with a case study that shows you what's possible, then explains exactly how to do it yourself. If you're new to Claude entirely, read the Getting Started section first — it covers setup, model selection, and the one technique that separates good Claude output from generic AI output.

What you'll need

  • Claude account — Pro plan minimum ($20/month). Max plan ($100/month) is recommended for heavy design work because it gives you more usage per day.
  • Claude Desktop app — the desktop app unlocks MCP connections. The web browser version does not support MCPs. Download it free from claude.ai.
  • A Figma account — Professional plan or above for MCP write access. Free plan is read-only.
  • Patience for the first session — the first time you connect Claude to a tool (like Figma), there's a setup step. After that, it's one click.
Important Note on Costs

This guide covers both Claude.ai (subscription) and the Claude API (pay-per-use). If you're just experimenting, stick to Claude.ai — your monthly fee covers everything. The API pricing section at the end of this guide is only relevant if you're building automated workflows in n8n or similar tools.

Getting Started

Universal Setup

Do this once. Every pillar in this guide builds on this foundation. It takes about 30–60 minutes for a careful first-timer.

Step 1 — Install Claude Desktop

  1. 1
    Go to claude.ai/download and download the Claude Desktop app for macOS (or Windows). This is different from the Claude website — the app is what enables MCP connections.
  2. 2
    Sign in with your Claude account. If you only have a free account, this is the moment to upgrade to Pro or Max.
  3. 3
    Open Settings (the gear icon, bottom-left). Under the Features tab, turn on: Code execution and file creation, Artifacts, and Search and reference past chats.

Step 2 — Create a Project for your work

Projects are like dedicated workspaces inside Claude. When you create a Project, Claude remembers your preferences, your brand rules, and your previous conversations within it. This is important — without a Project, Claude starts from scratch every session.

  1. 1
    In Claude Desktop, click New Project in the left sidebar.
  2. 2
    Name it something like "2X Design Work" or your specific project name.
  3. 3
    In the Project Instructions box, paste a short description of your brand context. For 2X work, something like: "You are working within 2X brand guidelines. Primary colours: Red #FF0F0F, Dark Gray #151515, White #FFFFFF. Typography: Inter Variable. Brand tone: bold, innovative, approachable. Always reference these when producing design-related outputs."

Step 3 — Connect your first Connector

Connectors are one-click integrations that let Claude talk to other tools. You connect them in Claude.ai (the browser version) or Claude Desktop under Settings → Connectors.

Recommended order to connect:

  1. Google Drive — for brief intake, file sharing, and export. Free.
  2. Figma — for the UIUX and Creative pillars. Requires Figma Professional+ for write access.
  3. Adobe for creativity — for Creative and Video pillars. Requires an Adobe Creative Cloud subscription.
  4. Canva — for Creative pillar. Free to connect; some features need Canva Pro.
  5. Slack — for approval loops. Requires a paid Slack workspace.
What is a Connector?

A Connector is a pre-built, verified integration that lets Claude access another app on your behalf. When you connect Figma, Claude can read your Figma files and (with the right account level) write directly to them. You always control what Claude can access — you approve each connection via a login screen, just like any other "Sign in with Google" flow.

Your First HITL Gate

Before moving to your pillar, test one connector. Open Claude, connect Google Drive, and ask Claude to "list the five most recently modified files in my Google Drive." If Claude can see your files, your setup is working. If it can't, re-check that you're in Claude Desktop (not the browser version) and that you approved the Google Drive connection in Settings.

Getting Started

Anti-Slop Prompting

The single most important technique in this entire guide. Every designer who gets great results from Claude uses this. Every designer who gets generic, forgettable output doesn't.

What is "AI slop"?

When you ask Claude (or any AI) to design something without telling it your aesthetic preferences, it produces what it has seen most often in its training: Inter font, purple gradient on white, three equal-size cards in a row, and a generic hero section. This is "AI slop" — technically correct, visually boring, and instantly recognisable as machine-generated.

The solution is not a different tool. It is a more specific brief.

The four fixes

  1. 1
    Name your aesthetic family. "Minimalist" is vague. "Brutalist editorial, like a 1990s fashion magazine printed on newsprint" is specific. Give Claude something to aim for.
  2. 2
    Name the fonts you want AND the ones you don't. Example: "Use Clash Display for headings and Fraunces for body. Do not use Inter, Roboto, or Arial."
  3. 3
    Give a reference URL or describe a reference. "Similar to the visual direction of linear.app" or "Reference: [paste a Dribbble or Behance URL]." Claude can look at a URL and extract the aesthetic direction.
  4. 4
    List what you do NOT want. "Avoid gradients. No rounded cards. No stock-photo style imagery. No three-column equal-width grid."

The Anti-Slop Brief Block

Copy this block into your Project Instructions (or paste it at the start of any design session). Replace the values in brackets with your actual preferences.

You are producing design work for [CLIENT/PROJECT NAME]. Aesthetic direction: [e.g. "Clean, enterprise SaaS. Confident and quiet. No decorative elements."] Typography: Use [FONT NAME] for headings. Use [FONT NAME] for body. Never use Inter, Roboto, or Arial. Colour: Primary [HEX], secondary [HEX], accent [HEX]. Dominant colour plus one sharp accent only. References: [URL or description of visual references] Do NOT use: [list defaults to avoid — e.g. purple gradients, three-card grids, stock-photo heroes] Motion: One deliberate animation moment per page load. Nothing decorative.
For 2X Projects

Replace the above with 2X brand values: Red #FF0F0F, Dark Gray #151515, Inter Variable at the weights in our brand system. Reference 2X.marketing for visual direction. Avoid any colours outside the primary and secondary palette.

Getting Started

What is an MCP?

MCP stands for Model Context Protocol. It sounds technical — but the idea is simple: it is a connector that lets Claude reach into another application and do things there, not just talk about them.

The plain-English version

Without an MCP, Claude can only read and write text in your conversation. With an MCP connected to Figma, Claude can actually open your Figma file, read your layers, create components, and write design tokens — all from inside Claude, while you watch it happen in Figma.

Think of an MCP as a remote control. Claude is the brain. The MCP is the remote that lets the brain press buttons inside another app.

MCPs vs Connectors — what's the difference?

In Claude.ai, Connectors are pre-built, one-click MCPs that have been vetted by Anthropic. They're the easiest way to connect Claude to Figma, Canva, Adobe, or Google Drive — you connect via a login screen, no setup required.

MCPs (in Claude Desktop or Claude Code) are the same technology, but you can install them yourself — including community-built ones that aren't in the official directory. This is what you use to connect to tools like Jumper (video editing) or Descript that aren't yet in the Connector directory.

For Beginners

Start with Connectors. They're in Settings and work with one click. Only look at custom MCPs once you've built a few weeks of experience with the Connector-based workflows in this guide.

How to install a custom MCP (when you need one)

This is covered in each pillar's Tool Stack section. The general process is:

  1. Open Claude Desktop → Settings → Developer → Edit Config
  2. Add the MCP's connection details (usually a URL or an install command)
  3. Restart Claude Desktop
  4. The new tool appears in Claude's available tools list
Figma MCP

Figma MCP — The Official One

Made by Figma. The primary bridge between Claude Code and your Figma files. Reads your designs and can write back to the canvas.

What it does

  • Reads your Figma file structure — layers, components, variables, auto-layout, styles
  • Generates code from selected frames (works best with Code Connect set up)
  • Writes to the canvas — creates frames, components, variables, and text
  • Converts live web UIs into editable Figma layers ("Send this to Figma")
  • Creates FigJam diagrams from Claude prompts (flowcharts, Gantt charts)

How to connect it (3 steps)

  1. 1
    In Claude.ai: Go to Settings → Connectors → click "+" → search "Figma" → Connect. Sign in with your Figma account. Done — no install needed.
  2. 2
    In Claude Code (for the full write-to-canvas features): Run claude mcp add --transport http figma https://mcp.figma.com/mcp in your terminal. Or install the Figma plugin from the Figma Community page which sets this up for you.
  3. 3
    Authenticate: Claude will prompt you to log in to Figma the first time. Do this from the Figma Desktop app for the best results — not the browser.
Plan Requirements

Reading Figma files: works on all plans including Free. Writing to the canvas (creating layers, components, variables): requires a Figma Professional seat or above. The write-to-canvas feature is currently free during beta — pricing for it has not been confirmed yet.

16 tools available

The official Figma MCP gives Claude 16 tools including: design context export, variable definitions, Code Connect mappings, screenshots, design system search, frame-to-code generation, and write-to-canvas operations. For most designers in this guide, you'll use three or four of these regularly.

Figma MCP

Figma Console MCP

Made by Southleft — an independent studio, not Figma. Open-source, free, with 57+ tools versus the official 16. The power tool option.

What makes it different

The official Figma MCP talks to Figma through the REST API — which is read-only for design data. You can read a file, but you can't create a rectangle or change a fill colour directly.

Figma Console MCP runs a Desktop Bridge plugin inside Figma using the Plugin API. This means anything a Figma plugin can do, Claude can do — create nodes, modify properties, manage variables in bulk, listen for real-time events.

Key capabilities the official MCP doesn't have

  • Full design system extraction in one call — no need to navigate layer by layer
  • 57+ tools including 11 dedicated variable/token management tools
  • Batch operations — handle up to 100 variables per call (10–50× faster than individual calls)
  • figma_setup_design_tokens — creates an entire token system (collection, modes, all variables) in one single operation
  • Real-time awareness — Claude knows what layer you've selected, what changed, and what page you're on
  • Health scoring — analyses your design system for inconsistencies and flags them
  • Open source (MIT) — free, no API keys required

How to install

  1. 1
    Install the Figma Console MCP Desktop Bridge plugin from the Figma Community. This is the plugin that runs inside Figma and gives Claude write access.
  2. 2
    Find your Claude config file. On Mac: ~/Library/Application Support/Claude/claude_desktop_config.json. Open it in any text editor.
  3. 3
    Add the server entry shown below and save the file. Restart Claude Desktop.
{ "mcpServers": { "figma-console": { "command": "npx", "args": ["-y", "figma-console-mcp"] } } }
Where to Find Your Config File

Mac: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows: %APPDATA%\Claude\claude_desktop_config.json
If the file doesn't exist yet, create it. Full guide: github.com/southleft/figma-console-mcp

Creative — Case Study 2

6-Slide Carousel in 4 Minutes

A solo creative cut their Instagram carousel production time from 40 minutes to under 4 by combining Claude with the Canva connector.

Creative ✓ Verified Solo content creator — social-first agency

A designer producing weekly Instagram carousels was spending 40 minutes per post designing each slide. Using Claude + Canva connector, the same output now takes under 4 minutes.

The key lesson: Canva connector prompts must be more specific than regular Claude prompts. Canva renders once — it can't course-correct mid-design. If your brief is vague, the output is vague and you can't fix it without starting over.

The prompt that worked

Canva: Create six Instagram post designs for a carousel. Slide structure: - Slide 1: Title slide — headline "[YOUR HEADLINE]" - Slides 2–5: One topic per slide. Each has: a bold topic name as the headline, a two-sentence explanation, and one visual indicator (icon or graphic element) - Slide 6: Closing slide with CTA: "[YOUR CTA TEXT]" Visual direction: [e.g. dark tech-forward aesthetic, clean editorial, warm minimal] Colour palette: [YOUR HEX CODES] Typography: [FONT PREFERENCES]

What this unlocks at 2X

The same pattern works for any 2X campaign carousel: weekly thought leadership, event announcements, service spotlights. Set up a Canva Brand Kit with 2X colours and typography first — then every carousel Claude creates via the connector will automatically apply brand styling.

Pro Tip: Discover Your Connector's Capabilities

Before starting any Canva workflow, run this prompt: "List every Canva connector feature available in this chat, using only the actual exposed tool list as source of truth. Output a bullet point list, with a sample prompt for each feature and the Canva plan required." This tells you exactly what Claude can do before you start — no guessing.

Creative — Case Study 3

Batch Photoshop from a Template + CSV

An Adobe community developer proved you can feed Claude a Photoshop template and a data file, and have it generate a new PSD for every row — automatically.

Creative ✓ Verified Adobe developer experiment — Photoshop MCP workflow

Using the community-built adb-mcp tool (which connects Claude to Photoshop via a plugin), a designer demonstrated: one Photoshop template + one CSV file → Claude generates a unique PSD for every row of data. Variable names, product descriptions, prices — all swapped in automatically.

This is essentially data merge, but controlled by Claude. You describe what should change per row, and Claude handles the repetition.

Verdict: proof-of-concept. It worked, but was slow (Claude timed out twice on larger batches). Good for 20–50 assets; less reliable for 500+. The official Adobe for creativity connector (launched April 2026) handles lighter versions of this at the Adobe Express level without any setup overhead.

Adobe for creativity connector — what it actually does

The official connector (in Claude.ai Settings → Connectors → Adobe for creativity) covers 50+ tools across Photoshop, Lightroom, Illustrator, Firefly, Premiere, Express, InDesign, and Stock. Important caveat: independent testing shows it operates at Adobe Express level — not pro-grade Photoshop. A 3-minute video reframe "didn't even centre the subject" in one test.

Best-fit uses for creative designers:

  • Portrait retouching sequences (lighting, background blur, auto-straighten, crop) across Lightroom + Photoshop
  • Express social-asset design for quick brand-safe posts
  • Horizontal-to-vertical reframe for platform repurposing (rough pass)
  • Adobe Stock search and licensing: "Search Adobe Stock for minimalist desk-setup photos and license the top result into my assets"
Realistic Expectation

The Adobe connector is a capable rough-pass tool, not a precision editing environment. Use it to generate a starting point, then refine in the actual application. Don't demo it to a client as a Photoshop replacement — it isn't one.

Creative — Case Study 4

Brand Guidelines from Existing Assets

A 30-year agency veteran built a client's brand "single source of truth" in 15 minutes — from a website URL, existing decks, and a brand PDF.

Creative ✓ Verified Agency principal — nonprofit client rebrand project

An agency principal fed Claude Design a nonprofit's existing website URL, two old presentation decks, and a short brand brief PDF. Within 15 minutes, Claude had extracted the colour palette, typography hierarchy, tone principles, and core visual motifs into a structured brand system.

Caveats: the designer burned through about 90% of their Claude weekly usage allowance in 4–5 prompts (Claude Design uses Opus 4.7, which is expensive for image-heavy tasks). They also noted that for minimal brand inputs (logo + hex code), Claude produces "a reasonable guess, not a real system." The richer your input, the better the output.

The workflow

  1. 1
    Gather inputs first: the client website URL, any existing deck or PDF, logo files, and a 2–3 sentence brief about their tone and audience.
  2. 2
    Open Claude Design (claude.ai/design — Pro or Max plan required). Upload all inputs.
  3. 3
    Prompt: "Extract and systematise the visual identity from these materials. Output: colour palette with hex values, typography hierarchy, tone principles (3–5 rules), and core visual motifs. Format as a brand system reference."
  4. 4
    Review the output, correct any misinterpretations, then ask Claude to build the guidelines document: "Turn this into a brand guidelines PDF I can share with vendors."

How this works for 2X creative projects

For new client onboarding: paste the client's website URL + any existing brand assets into Claude at the start of a project. Ask it to extract the brand system. Use that as your brief foundation — then add your own creative direction on top. This alone can save an hour of manual brand audit work per new client.

Token Warning

Claude Design uses vision-heavy processing. If you're on the Pro plan, treat each brand-extraction session as a significant usage event. On Max plan, you get much more room. If you're hitting limits, do the brand extraction in Claude.ai (not Claude Design) with plain text descriptions of the brand — you get 80% of the output at 5% of the token cost.

Creative — Figma Ops

Reducing Repetitive Work in Figma

Some of the highest-value Claude workflows for creative designers don't involve generating new content at all — they're about eliminating the Figma housekeeping that eats your time.

Layer and frame renaming

Figma's native AI can rename layers (it's in the right-click menu — "Rename with AI"). But Claude + Figma MCP goes further: you can define your own naming convention and have every layer in a file renamed to match it in one pass.

Rename all layers in this Figma file according to this convention: - Frames: [PageName]/[SectionName]/[ComponentName] - Text layers: text/[purpose] e.g. text/headline, text/body, text/cta - Image layers: img/[description] - Background layers: bg/[colour or description] Only rename layers that still use Figma's default naming (Frame 1, Rectangle 2, etc.). Keep any layers I've already renamed.

Text and content insertion for digital ads

If you're running digital ad campaigns in Figma, Claude + Figma MCP can insert headline, body copy, and CTA text into all your frames from a brief — so you're not manually pasting text into 12 ad sizes.

In this Figma file, I have ad frames for sizes: 300×250, 728×90, 160×600, 320×50. For each frame, insert this copy: Headline: [YOUR HEADLINE — max 40 chars] Body: [YOUR BODY COPY — max 90 chars] CTA button: [YOUR CTA — max 20 chars] Place headline in the layer named "text/headline", body in "text/body", and CTA in the button layer named "btn/cta".

Figma Buzz + Claude (Data Merge)

Figma Buzz is a Figma plugin for data merge — feeding spreadsheet data into a Figma template to generate multiple versions. Claude acts as the intelligent layer on top: it writes the content that goes into the spreadsheet, validates it against character limits, and can even suggest which template variations to create.

Workflow:

  1. Set up your Figma template with named text layers (matching your Buzz column names)
  2. Ask Claude to generate the data file: "I have a Figma Buzz template with columns: Name, JobTitle, Company, Headline. Generate 20 rows for [CAMPAIGN/EVENT NAME], keeping Headline under 60 characters."
  3. Export Claude's output as CSV → import into Figma Buzz → generate all variants

UI/UX — Case Study 3

Stopping Claude from Breaking Your Design System

The most common frustration with Claude + Figma: it writes a hardcoded colour (#5C6AC4) instead of a token reference (color/brand/primary). A UIUX designer built a four-step harness that fixes this permanently.

UI/UX ✓ Verified Design systems practitioner — token enforcement workflow

The problem: Claude would produce visually correct output (the right colour) but structurally wrong output (a hardcoded hex value instead of a token reference). This passes visual QA but breaks the design system — developers consume the tokens, not the hex values.

The solution: a four-step Skill harness that forces Claude to check its work before and after each operation.

The four-skill harness

  1. 1
    Preflight Skill — before Claude creates any node, it must output a Token Map (every token name and its value) and a Component Registry (every component in the file). Claude cannot proceed until this passes.
  2. 2
    Design Brief Parser — structures the incoming request against the registered tokens. Any colour, spacing, or radius reference in the brief is mapped to a token name before any Figma operation begins.
  3. 3
    Write to Figma — executes with token references enforced. Claude uses token names (color/brand/primary) not hex values.
  4. 4
    QA + Rebinding Skill — audits the output. Finds any hardcoded values and triggers a rebinding pass to replace them with the correct token references.

A simpler version for beginners

If you're not ready to build a full Skill harness, add this line to every Figma prompt:

Important: do not hardcode any colour, spacing, or border radius values. Every value must reference a named token from the variables panel. If a token doesn't exist for the value you need, ask me before proceeding.

UI/UX — Claude Design

Claude Design for UI/UX

Claude Design is Anthropic's prompt-to-prototype tool. It's fast for upstream exploration and struggling for sustained high-fidelity work. Here's when to use it — and when not to.

What Claude Design does

You describe what you want, Claude generates a working prototype (HTML/CSS/JS) that you can see in a browser. It can also read your codebase or Figma file to extract your design system, then generate designs that use your actual tokens and components. When you're happy with a direction, it hands off to Claude Code for production.

Use it for

  • Rapid direction exploration — get 3 directions in 15 minutes, then choose one
  • Quick stakeholder mockups before committing to Figma
  • Extracting a design system from a clean codebase
  • WCAG accessibility self-audits: "Review this design for WCAG 2.1 AA contrast and accessibility violations with specific fixes."
  • Turning a rough PRD or spec into a visual wireframe for stakeholder review

Avoid it for

  • Sustained high-fidelity design work — Figma is better for that
  • Any task where you're on Pro plan and have limited usage — Claude Design burns tokens fast
  • Design systems built from minimal brand input (logo + hex code = "reasonable guess")
⚠️ Token Warning

Claude Design uses Opus 4.7, which charges roughly 3× more per image token. Multiple users on the Pro plan burned through their entire weekly allowance in one 30-minute session. If you're on Pro, treat each Claude Design session as a significant event. On Max plan, you have much more room.

The three-direction exploration prompt

Show me 3 different directions for [COMPONENT/PAGE NAME]. Direction 1: Minimal — clean, lots of white space, typography-led Direction 2: Expressive — bold visuals, stronger colour use, more personality Direction 3: [YOUR CUSTOM DIRECTION — e.g. retro-technical, editorial, data-dense] Use [YOUR BRAND SYSTEM] tokens throughout. Do not repeat the same layout across directions.

Review this design for WCAG 2.1 AA compliance. Check: - Colour contrast ratios (minimum 4.5:1 for text, 3:1 for large text and UI) - Interactive element sizes (minimum 44×44px touch target) - Focus states on all interactive elements - Alt text requirements for images - Heading hierarchy For each issue: describe the problem, the WCAG criterion it violates, and the specific fix.

Fetch this Figma design file, read the README, and generate developer handoff documentation. UX critique

Review this design from a UX perspective. Evaluate: - Information hierarchy — is the most important thing visually dominant? - Friction points — where might a user get stuck or confused? - Missing states — what happens when the list is empty? When loading? On error? - Cognitive load — is there too much on screen at once? Give specific, actionable feedback. Describe what you'd change and why. Do not tell me it "looks great." Reference

Glossary

Plain-English definitions for every technical term in this guide.

MCP (Model Context Protocol)

A connector that lets Claude reach into another application and perform actions there. Like a remote control that lets Claude press buttons inside Figma, Descript, or Canva.

Connector

A pre-built, verified MCP inside Claude.ai that you connect with one click via a login screen. Figma, Canva, and Adobe for creativity are Connectors.

Skill

A packaged set of instructions you install into Claude once, and it applies every session. Like a saved preference that makes Claude always follow your brand rules.

HITL (Human in the Loop)

Every workflow in this guide has a point where a human reviews before anything is published or shared. That's the HITL gate — you approve, then Claude continues.

Token

The unit Claude uses to measure text. Roughly 1 token = 0.75 words. Relevant for API (pay-per-use) billing. Not relevant for Claude.ai subscription users.

Design Token

A named design decision stored as a variable — e.g. color/brand/primary = #FF0F0F. The name is used in both Figma and code, so design and engineering stay in sync.

Semantic Token

A design token named by purpose (color/semantic/danger) rather than value (color/primitive/red/500). Semantic tokens reference primitive tokens — so changing a primitive updates all semantic tokens automatically.

Claude Code

A terminal-based version of Claude that can run code, install tools, and perform complex multi-step operations. Used for Figma MCP write operations, design system builds, and n8n workflow creation.

Claude Design

Anthropic's prompt-to-prototype tool (launched April 2026, in research preview). You describe a UI; it generates a working prototype. Best for rapid upstream exploration, not sustained production work.

n8n

An open-source automation platform — like Zapier or Make, but self-hostable and free. Connects Claude to your other tools and schedules automated workflows.

Agentic workflow

A Claude session where Claude takes multiple steps autonomously — deciding what to do next based on the results of the previous step — rather than waiting for a new prompt each time.

Code Connect (Figma)

A Figma feature that maps your Figma components to your actual code components. When enabled, Claude generates code using your real imports — not generic HTML.

Underlord

Descript's built-in AI editing assistant. When you use Claude + Descript MCP, Claude can instruct Underlord to remove fillers, apply Studio Sound, generate captions, and more.
